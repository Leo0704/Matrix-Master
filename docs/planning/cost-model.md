### 3.6 成本优化

- **Haiku 用于 review / schedule**：质量可接受，成本 -70%
- **缓存 prompt**：相同 prompt 缓存（Anthropic prompt caching），节省 50%
- **批处理**：多条 run 合并 LLM 调用
- **本地小模型兜底**：超 budget 降级到本地 7B 模型

## 7. 商业化

### 7.1 单设备回本估算

- 假设：单设备日均涨粉 30，3 个月到达 2700 粉
- 商业价值：单粉价值 ¥0.5（接广告 / 带货）
- 单设备 3 个月收入：2700 * 0.5 = ¥1,350
- 单设备 3 个月成本：¥22,720 / 100 * 3 = ¥681
- **单设备 3 个月净利：¥669**

### 7.2 规模化回本

- 100 设备 3 个月净利：¥67,000
- 减去人力后：¥67,000 - ¥24,000 = ¥43,000

### 7.3 关键假设

以上估算依赖：
- 涨粉率（最关键）
- 单粉价值（电商 / 广告市场）
- 账号存活率（封号风险）
- 设备养号成本（人力）

## 8. 成本控制策略

### 8.1 短期

- 用 Haiku 替代 Sonnet 做简单任务
- 启用 prompt caching
- 监控 LLM 调用异常

### 8.2 中期

- 本地 7B 模型做兜底
- VLM 调用前选择器 cache
- 多设备复用同一 LLM 调用结果

### 8.3 长期

- 微调本地模型（基于运营数据）
- 自托管 embedding + 简单 VLM

## 9. 成本监控

每日成本看板：
- LLM 成本（按模型 / 按 call_type）
- VLM 成本
- VPS 成本（固定）
- 移动数据成本（固定）

告警：
- LLM 日成本 > 预算 → critical
- LLM 单 run 成本 > $1 → warning
- VLM 日成本 > 预算 → critical

详见 [monitoring-runbook.md §2.5](../operations/monitoring-runbook.md)。
