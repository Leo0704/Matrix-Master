# 威胁模型（Threat Model）

| 项 | 内容 |
|---|---|
| 版本 | v0.1 |
| 主要读者 | 安全负责人 / 架构师 / 开发 lead |
| 配套文档 | [SDD.md](./SDD.md) / [PRD.md](../../PRD.md) |

本文档识别系统面临的威胁，定义缓解措施与责任分工。**新增功能 / 接口时必须更新本文档**。

---

## 1. 范围

### 1.1 在范围内
- 主控桌面应用（Tauri shell + Python 后端）
- 手机端 companion APK
- Tailscale mesh 控制通道
- 知识库与业务数据（PostgreSQL）
- 账号凭据
- LLM / VLM 集成
- 运营者与系统的交互

### 1.2 不在范围
- 小红书 App 自身的安全机制
- 手机操作系统（Android）自身的安全机制
- 物理设备被盗 / 丢失
- 内部恶意员工（假设为可信）

## 4. 攻击面

### 4.1 攻击面清单

| ID | 攻击面 | 入口 | 描述 |
|---|---|---|---|
| A1 | 主控↔APK HTTP API | Tailscale mesh | 任意 tailnet 节点可尝试调 |
| A2 | Tailscale 控制面 | Headscale 节点 | 可注册新设备、查询 ACL |
| A3 | LLM prompt | LLM API | 用户输入 / 知识库内容被注入 |
| A4 | 知识库内容 | PostgreSQL | 运营者写入恶意内容 |
| A5 | 配置文件 | 本地文件 | API key / 密钥明文泄露 |
| A6 | 设备物理接触 | 手机本身 | 物理拿到手机可提取 APK 数据 |
| A7 | 数据库备份 | 备份文件 | 备份含敏感数据 |
| A8 | 主控 UI 渲染 | Web 前端 | XSS / 注入 |
| A9 | 任务调度 | asyncio 队列 | 内部攻击者注入恶意 task |
| A10 | 监控/日志导出 | 日志文件 | 含敏感数据 |

## 5. 威胁分析（STRIDE 分类）

### 5.1 Spoofing（伪装）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| 伪 APK 调主控 | 攻击者在 tailnet 伪造 APK 调主控 API | HMAC 鉴权 + 设备白名单（`devices.status = 'active'`） | 后端 |
| 伪主控调 APK | 攻击者控制主控后下发恶意指令 | APK 端操作白名单（仅允许文档中定义的操作） | APK |
| 设备伪装 | 攻击者换 IP 模拟已注册设备 | Tailscale 节点身份验证 + HMAC | 后端 |

### 5.2 Tampering（篡改）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| 知识库内容被改 | 注入违规内容导致违规发布 | persona / rule 写入需人工 review | 运营 + 后端 |
| 任务 payload 篡改 | 攻击者改 task 发恶意内容 | request_id 校验 + 设备绑定校验 | 后端 |
| APK 二进制被改 | 重打包 APK 加入恶意代码 | APK 签名校验（主控侧校验 APK 签名哈希） | 后端 |
| PostgreSQL 数据改 | 直接改库 | 数据库访问控制（仅主控后端账号） | 运维 |
| 备份被改 | 备份文件被替换 | 备份加密 + 完整性校验 | 运维 |

### 5.3 Repudiation（抵赖）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| 运营者否认操作 | 误操作后抵赖 | 所有写操作审计日志（操作者 + 时间 + 内容） | 后端 |
| APK 否认执行 | APK 报失败但实际成功 | APK 上报必须含截图 + 时间戳，主控独立校验 | APK + 后端 |
| 管理员否认配置改动 | 误配置后抵赖 | 配置文件改动走 Git + 审计 | 运维 |

### 5.4 Information Disclosure（信息泄露）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| 凭据泄露 | XHS 账号密码 / token 被提取 | Android Keystore 加密 + 主控 keyring；日志中**禁止打印**凭据 | APK + 后端 |
| HMAC 密钥泄露 | 攻击者提取密钥伪造指令 | Keystore 加密 + 永不出 APK 进程；定期轮换 | APK |
| LLM API key 泄露 | 配置文件被读 | 文件权限 600 + 系统 keyring | 运维 |
| 数据库泄露 | 数据库文件被拷 | 全盘加密（LUKS / FileVault）+ 网络隔离 | 运维 |
| 备份泄露 | 备份文件外流 | 备份加密 + 异地存储访问控制 | 运维 |
| 截图泄露 | device_collect 返回的截图含敏感信息 | 截图本地处理，VLM API 调用前脱敏（裁剪个人头像） | APK + 后端 |
| 监控面板泄露 | 内部监控被外部访问 | 监控仅本地暴露，不对外开放端口 | 运维 |
| 日志泄露 | 日志文件含敏感信息 | 日志脱敏（密码 / token 替换为 `***`） | 后端 |

### 5.5 Denial of Service（拒绝服务）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| 主控被刷 | 大量请求压垮主控 | API 限速 + 鉴权失败熔断 | 后端 |
| Headscale 被打 | 控制面过载 | Headscale 部署在受保护 VPS + 防火墙 | 运维 |
| LLM API 限流 | 触发 LLM provider rate limit | 限速器 + 重试 + 降级到本地模型 | 后端 |
| 数据库写满 | 日志 / 监控数据爆盘 | 监控数据保留期 + 自动清理 | 运维 |
| Tailscale DERP 失联 | 国内访问官方 DERP 不稳 | 自建 DERP（Headscale） | 运维 |
| 设备掉线 | 蜂窝信号 / 手机故障 | 限速 + 等待恢复 + 告警 | 运营 |

### 5.6 Elevation of Privilege（权限提升）

| 威胁 | 描述 | 缓解 | 责任 |
|---|---|---|---|
| APK 提权 | APK 借 AccessibilityService 越权 | APK 签名 + 仅响应白名单 tool + 操作范围限制 | APK |
| 运营者提权 | 普通运营者调管理接口 | RBAC（管理员 / 运营 / 只读三档） | 后端 |
| 数据库提权 | 业务账号获取超级用户权限 | 业务账号最小权限；超级用户账号独立 + 强密码 | 运维 |
| LLM 提权 | prompt injection 让 LLM 执行系统命令 | Agent 工具白名单 + LLM 输出严格解析 | 后端 |
| 前端 XSS | Web 前端执行恶意脚本 | 前端 CSP + 输入转义 + 不渲染用户 HTML | 前端 |

## 6. 专项威胁深入

### 6.1 LLM Prompt Injection

**威胁**：运营者或外部输入被注入到 LLM prompt，导致 Agent 执行非预期操作。

**攻击向量**：
- 知识库内容（persona / rule / topic）被污染
- 抓取的热点内容含 prompt 注入
- LLM 返回的内容中含 prompt 注入链

**缓解**：
1. LLM 输入清洗：剥离 markdown / HTML 标签，仅保留纯文本。
2. 工具调用白名单：LLM 只能调用预定义的 tool，不能任意执行。
3. tool 输入严格 schema 校验：LLM 输出的参数必须符合 schema。
4. 敏感操作二次确认：发布 / 互动类写操作经规则引擎校验后才执行。
5. 异常行为监控：LLM 调 tool 频次异常时熔断。

**责任**：后端（实现校验 + 监控）

### 6.2 账号凭据保护

**威胁**：XHS 账号被他人获取。

**存储**：
- 主控：不存储明文凭据；如需存储（自动登录态），用 OS keyring（macOS Keychain / Windows Credential Manager）加密。
- APK：登录态由 XHS App 自身管理；APK 不保存凭据。

**传输**：
- 主控↔APK 所有调用经 Tailscale 加密。
- 主控↔LLM API 经 HTTPS。
- APK↔XHS App 经 Android IPC + XHS 自身 TLS。

**日志**：
- 任何日志中**禁止**打印凭据、token、cookie。
- 代码 review 时强制检查。

**责任**：后端 + APK + 运维

### 6.3 HMAC 密钥生命周期

**生成**：主控首次添加设备时生成（256 bit 随机）。
**下发**：经 Tailscale 通道，仅一次，下发后立即从主控临时内存清除。
**存储**：
  - 主控：配置文件 + 权限 600。
  - APK：Android Keystore 加密。
**轮换**：每月一次自动轮换，运营者无需介入。
**撤销**：设备下线时立即从主控配置中删除。
**泄露应急**：发现泄露立即轮换 + 通知所有相关设备重新配对。

**责任**：后端 + APK

### 6.4 备份安全

**加密**：所有数据库备份用 AES-256-GCM 加密，密钥由运维主密钥派生。
**存储**：本地 + 异地（OSS / S3），异地访问需 MFA。
**保留**：本地 7 天，异地 30 天。
**恢复演练**：每季度一次恢复演练。
**完整性**：备份含 SHA-256 校验和，恢复后校验。

**责任**：运维

### 6.5 审计日志

**记录内容**：
- 谁（user_id）何时（ts）做了什么（action）改了什么（before/after）。
- 所有写操作（创建 / 更新 / 删除）。
- 所有认证事件（登录 / 失败 / 密钥轮换）。

**不可篡改**：审计日志追加写入专用文件，文件权限 0400，定期归档到只读存储。

**保留**：1 年。

**责任**：后端 + 运维

## 9. 应急响应

### 9.1 账号被盗用

1. 立即在监控控制台暂停该账号。
2. 通过 XHS 平台冻结账号 / 修改密码。
3. 排查泄露路径（凭据存储 / 中间人 / 设备丢失）。
4. 更新本威胁模型，加新缓解。

### 9.2 HMAC 密钥泄露

1. 立即轮换主密钥（强制所有 APK 重新配对）。
2. 审查日志确认泄露时间窗口。
3. 检查是否被用于未授权操作。
4. 评估是否需要回滚部分数据。

### 9.3 设备丢失

1. 在主控控制台下线该设备（删除 `devices` 记录 + 吊销 Tailscale 节点）。
2. 远程擦除 APK 数据（如设备支持）。
3. 评估该设备账号是否被访问。

### 9.4 数据库泄露

1. 立即轮换所有密钥。
2. 评估泄露数据范围。
3. 通知受影响方（运营者 + 用户）。
4. 启动事件响应流程（含监管报告义务评估）。
