# 故障复盘模板

| 项 | 内容 |
|---|---|
| 适用对象 | on-call / 团队 lead |
| 配套 | [monitoring-runbook.md](./monitoring-runbook.md) |

故障发生后 1 周内完成复盘。**blameless**：只追系统原因，不追个人责任。

## 故障概要

| 项 | 内容 |
|---|---|
| 故障 ID | INC-YYYYMMDD-NNN |
| 故障标题 | 一句话描述 |
| 严重度 | P0 / P1 / P2 |
| 开始时间 | YYYY-MM-DD HH:MM UTC |
| 结束时间 | YYYY-MM-DD HH:MM UTC |
| 总时长 | X 小时 Y 分钟 |
| 受影响范围 | 多少设备 / 账号 / 用户 |
| 负责人 | 姓名 |

## 时间线

按时间倒序或正序记录关键事件：

| 时间 (UTC) | 事件 | 操作人 |
|---|---|---|
| HH:MM | 告警触发 | 系统 |
| HH:MM | on-call 收到通知 | 姓名 |
| HH:MM | 初步判断 | 姓名 |
| HH:MM | 升级到团队 lead | 姓名 |
| HH:MM | 实施缓解 | 姓名 |
| HH:MM | 故障恢复 | - |
| HH:MM | 监控确认 | 姓名 |

## 影响

- **服务可用性**：X% 用户受影响
- **数据丢失**：是 / 否，详情
- **资金损失**：估算（如有）
- **用户感知**：投诉 / 反馈数
- **业务影响**：未能发布的笔记 / 错过的互动
